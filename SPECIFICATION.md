# Wellbeing — Технічне завдання та специфікація проєкту

> Версія: 1.0 | Дата: 2026-05-11

---

## 1. ЗАГАЛЬНИЙ ОПИС ПРОЄКТУ

**Wellbeing** — корпоративна платформа психологічної та загальної підтримки співробітників. Система дозволяє компаніям надавати своїм співробітникам доступ до спеціалістів (психологів, юристів, коучів, нутриціологів тощо), управляти корпоративними підписками, відстежувати стан користувачів та автоматизувати запис на консультації.

### 1.1 Учасники системи

| Роль | Опис |
|---|---|
| **Користувач** | Співробітник компанії, який записується на консультації |
| **Адміністратор** | Оператор платформи з повним доступом до всіх модулів |
| **Спеціаліст** | Консультант з обмеженим кабінетом (свої записи і розклад) |

### 1.2 Технологічний стек

| Шар | Технологія |
|---|---|
| **Frontend** | Angular 17+, TypeScript, ngx-translate (i18n), CSS custom properties |
| **Backend** | PHP 8.x, Yii2 (REST), PostgreSQL (Supabase) |
| **Auth** | JWT Bearer Token |
| **Деплой** | GitHub Actions CI/CD, rsync, systemd, Caddy |
| **Кешування** | Yii2 FileCache / MemCache |
| **БД** | PostgreSQL (Supabase), міграції через Yii2 Console |

## 2. КАБІНЕТ КОРИСТУВАЧА

### 2.1 Авторизація та реєстрація

#### Вхід (`/login`)
- Форма: email + пароль
- JWT Bearer авторизація, зберігання токену в localStorage
- Редирект на `/dashboard` після входу
- Посилання на реєстрацію
- Захист: Google reCAPTCHA v3

#### Реєстрація (`/register`)
- Поля: ім'я, прізвище, email, компанія (dropdown), пароль
- Компанія: динамічний список з БД (`GET /v1/company`)
- Автогенерація надійного пароля (кнопка ✦ у полі)
- Показати/приховати пароль (eye toggle)
- Підписка на умови використання та політику конфіденціальності (checkbox)
- Модальне вікно з повними текстами умов та політики (WYSIWYG контент)
- Захист: Google reCAPTCHA v3
- Після реєстрації — автоматичний вхід + редирект на `/dashboard`

#### Вихід
- Кнопка у сайдбарі ("Вийти")
- Очищення токену, редирект на `/login`

### 2.2 Дашборд (`/dashboard`)

#### Статистика (4 картки)
- Запланованих консультацій
- Завершених консультацій
- Залишок безкоштовних сеансів
- Скасованих
- Анімація лічильників при завантаженні (ease-out cubic)

#### Корпоративні безкоштовні сеанси
- Прогрес-бар: використано / ліміт (місячний)
- Лімит задається на рівні компанії (`free_sessions_per_user`)
- Якщо вичерпано — повідомлення про платний режим
- Реальний час: підрахунок по поточному місяцю

#### Трекер настрою (mini-chart)
- 7 останніх днів, квадратики з кольором та emoji за рівнем (1–5)
- Клікабельний — переходить на `/questionnaire`

#### Найближчі записи
- Список 3 найближчих підтверджених/очікуючих записів
- Ім'я спеціаліста, спеціалізація, дата/час, статус-бейдж
- Посилання "Всі записи →"

#### Google Calendar (блок)
- Стан "не підключено": кнопка підключення Google Calendar
- Стан "підключено": список найближчих подій з Google Calendar
  - Лише події з міткою "Wellbeing" у description
  - Назва, дата/час, кнопка "Meet" (якщо є посилання)
- Підпис "Підключено" (зелений бейдж)
- Авто-оновлення кожні 60 секунд

---

### 2.3 Мої записи (`/appointments`)

#### Список записів
- Картки: фото/ініціали спеціаліста, ПІБ, спеціалізація, дата, час, статус
- Статус-бейджі: Підтверджено / Очікування / Завершено / Скасовано / Не з'явився
- Рядок оплати: іконка + статус (Оплачено / Корпоративна підписка / Очікує оплати / Повернено / Не оплачено)
- На мобілці: дата переноситься на новий рядок

#### Фільтри
- Всі / Майбутні / Завершені / Скасовані / Оплачені
- Фільтрація на фронтенді (без API запитів)

#### Дії над записом
- **Оплатити** (якщо статус pending)
  - Ініціює checkout через платіжний шлюз
  - Редирект на сторінку оплати та назад з результатом
  - Toast-сповіщення: успіх / помилка
- **Скасувати** (якщо confirmed/pending)
  - Модальне вікно підтвердження
  - Попередження про повернення коштів (якщо оплачено)
  - Поле "Причина скасування" (обов'язкове якщо < 2 год до початку)
  - Toast: скасовано / повернено / помилка повернення
- **Залишити відгук** (якщо completed і немає відгуку)
  - Модальне вікно: 5 зірок + текстовий коментар
  - Попередження при оцінці 1–2: "Наш оператор зв'яжеться з вами"
  - Блокування повторного відгуку

---

### 2.4 Бронювання (6-крокове модальне вікно)

Відкривається з `/appointments` (кнопка "Новий запис") або з сайдбару (кнопка "Записатись").

#### Крок 1: Каталог спеціалістів
- Фільтр **Спеціалізація** (чіпи): Всі / Психолог / Юрист / Коуч / та ін.
  - Динамічно з унікальних типів завантажених спеціалістів
- Фільтр **Категорія** (чіпи): показується лише якщо є категорії для обраної спеціалізації
  - При зміні спеціалізації — скидається на "Всі категорії"
- Картка спеціаліста: фото/ініціали, ПІБ, тип, ціна (₴), категорії, досвід, рейтинг/кількість відгуків
- **Закріплений спеціаліст**: якщо у користувача вже є записи — цей спеціаліст показується першим з бейджем "✓ Ваш спеціаліст" та зеленою лівою рамкою
- Порожній стан

#### Крок 2: Профіль спеціаліста
- Аватар, ПІБ, тип, досвід
- Повний опис (bio)
- Відгуки клієнтів: зірки, коментарі, дата
- Кількість відгуків
- Вибір способу оплати: Онлайн картою / Корпоративна підписка
  - Підписка доступна якщо є залишок безкоштовних сеансів у компанії
  - Якщо ліміт вичерпано — статус "Ліміт вичерпано"

#### Крок 3: Вибір слоту
- Підказка: оберіть день і час
- Список доступних дат (14 днів вперед)
- При виборі дня — список доступних часових слотів
- Виключені: заблоковані дні спеціаліста, вже заброньовані слоти, минулі слоти
- Відображення на кирилиці (українські місяці)

#### Крок 4: Деталі запису
- Вибір способу зв'язку: Google Meet / Zoom / Microsoft Teams
  - Для кожного — іконка та підпис
- (Вибір способу оплати — якщо не вибрано на кроці 2)

#### Крок 5: Підтвердження
- Підсумок: спеціаліст, дата/час, спосіб зв'язку, вартість
- Блок оплати: Підписка / Онлайн-оплата / Безкоштовно
- Перша доступна дата та час у полі
- Перейти до підтвердження

#### Крок 6: Успіх
- Повідомлення "Запис підтверджено!"
- Підказка про email-підтвердження та нагадування за 12 год
- Якщо оплата pending: кнопка "Оплатити зараз" + посилання на записи
- Кнопка "Закрити"

#### Навігація між кроками
- Прогрес-бар зі статусами кроків
- Кнопки: "Відмінити" (крок 1) / "← Назад" / "Далі →" / "Підтвердити запис"
- Кнопка "Далі" неактивна якщо крок не заповнений

### 2.5 Профіль (`/profile`)

#### Блок аватару
- Відображення фото або ініціали на градієнтному фоні
- Завантаження фото (кнопка "Змінити фото")
  - Overlay з спіннером під час завантаження
  - Повідомлення про помилку
- ПІБ, компанія, ID користувача
- Кнопка "Змінити пароль" → відкриває модальне вікно

#### Форма особистих даних
- Поля: Прізвище, Ім'я, Телефон, Email (не редагується), Компанія (не редагується)
- Режим "тільки читання" за замовчуванням
- Кнопка "Редагувати" → всі поля стають активними
- Кнопки "Скасувати" / "Зберегти" (з спіннером)
- Повідомлення про успішне збереження / помилку

#### Зміна пароля (модальне вікно)
- Поля: поточний пароль, новий пароль, підтвердження
- Eye toggle для кожного поля
- Автогенерація пароля (кнопка ✦ у полі нового пароля)
- Анімований вхід/вихід (fade + slide)
- Валідація: збіг паролів, мінімальна довжина
- Повідомлення про помилку / успіх
- Закриття на кнопку ✕ або клік по оверлею

#### Умови використання
- Чекбокс "Я погоджуюсь з умовами..."
- Клікабельні посилання → модальне вікно з повним текстом (WYSIWYG)
- Зміна зберігається автоматично (без збереження форми)

- Лічильник непрочитаних (бейдж у сайдбарі)
- Порожній стан

### 2.9 Анкета / Трекер настрою (`/questionnaire`)

#### Трекер настрою
- Щоденна оцінка настрою: шкала 1–5 (дуже погано → відмінно)
- Збереження через API (одна оцінка на день)
- Стан "вже збережено сьогодні"
- Emoji-ілюстрації до кожного рівня

#### Опитування (surveys)
- Список активних опитувань від адміністратора
- Покрокове проходження: кілька сторінок питань
- Типи питань: одиночний вибір, множинний вибір, текст
- Стан "вже пройдено" (блокує повторне проходження)
- Стан "немає активних опитувань"
- Надсилання результатів через API

  - Спосіб зв'язку (dropdown)
  - Повідомлення (textarea)

### 2.11 Налаштування (`/settings`)

#### Мова інтерфейсу
- Dropdown вибору: Українська / English / Русский

- Повна локалізація всього інтерфейсу

#### Вкладка "Загальні" (сповіщення)
- Email сповіщення (toggle): підтвердження і нагадування на пошту

  - Кнопка "Підключити Google Calendar" → OAuth redirect
  - Якщо підключено: кнопка "Відключити" + email
  - Повне управління токенами (refresh, revoke)

## 3. АДМІН-ПАНЕЛЬ

### 3.1 Вхід адміністратора (`/admin/login`)
- Окрема форма входу
- Перевірка ролі `is_admin`
- Редирект на `/admin/dashboard`

---

### 3.2 Дашборд адміністратора (`/admin/dashboard`)

- Загальна статистика: кількість записів, компаній, спеціалістів, користувачів
- Графік або таблиця активності
- Швидкий доступ до ключових розділів

---

### 3.3 Управління компаніями (`/admin/companies`)

- Список компаній: назва, кількість користувачів, ліміт сеансів, статус
- Створення компанії:
  - Назва, ліміт безкоштовних сеансів на місяць, домен email
  - Завантаження логотипу (зберігається у S3/storage)
  - Налаштування кольору бренду (primary_color)
  - Creatio ID компанії для синхронізації
- Редагування / видалення компанії
- Прив'язка логотипу через upload endpoint

---

### 3.4 Управління користувачами (`/admin/users`)

- Список користувачів: ПІБ, email, компанія, роль, дата реєстрації
- Фільтрація по компанії
- Створення/редагування/видалення користувача
- Зміна компанії користувача
- Скидання пароля
- Перегляд записів конкретного користувача

---

### 3.5 Управління спеціалістами (`/admin/specialists`)

- Список спеціалістів: фото, ПІБ, тип, категорії, ціна, статус (активний/неактивний)
- Створення спеціаліста:
  - ПІБ, тип (зі списку спеціалізацій), bio, досвід (роки), ціна (грн)
  - Категорії (мультивибір)
  - Завантаження аватару
  - Email спеціаліста (для Calendar-інвайтів)
  - Прив'язка до акаунту користувача (для кабінету спеціаліста)
  - Creatio Contact ID
- Редагування / видалення
- Активація / деактивація

#### Розклад спеціаліста
- Вибір днів тижня та часових слотів (сітка по годинах/30 хв)
- Збереження розкладу через API
- Поточний тижневий графік

#### Блокування дат
- Блокування конкретних дат (відпустка, відсутність)
- Розблокування

---

### 3.6 Управління спеціалізаціями (`/admin/specializations`)

- Список спеціалізацій: ключ (key), назва
- Ключ (key) використовується як `specialist.type`
- CRUD: створення, редагування, видалення
- Назва відображається у каталозі та картці спеціаліста

---

### 3.7 Управління категоріями (`/admin/categories`)

- Список категорій: назва, статус (активна/неактивна)
- CRUD: створення, редагування, видалення, зміна статусу
- Категорії відображаються як теми (Тривога та стрес, Депресія, тощо)
- Прив'язуються до спеціалістів (через поле `categories` у спеціалісті)

---

### 3.8 Управління записами (`/admin/appointments`)

- Таблиця всіх записів: користувач, спеціаліст, дата/час, статус, оплата, спосіб зв'язку
- Фільтрація по статусу, дату, спеціалісту, компанії
- Зміна статусу запису вручну
- Видалення запису (адмін)
- Перегляд деталей

---

### 3.9 Управління оплатами (`/admin/payments`)

- Таблиця всіх транзакцій: користувач, запис, сума, шлюз, статус, дата
- Фільтрація
- Ручна зміна статусу оплати
- Перегляд raw response від платіжного шлюзу

---

### 3.10 Управління слотами (`/admin/slots`)

- Перегляд розкладу всіх спеціалістів у вигляді сітки
- Вибір спеціаліста + редагування розкладу
- Блокування дат для спеціаліста

---

### 3.11 Опитування (`/admin/surveys`)

- Список опитувань: назва, статус (активне/неактивне), кількість відповідей
- Створення/редагування опитування:
  - Назва, опис
  - Додавання питань: текст, тип (single/multiple/text), варіанти відповідей
- Активація/деактивація опитування (одночасно може бути одне активне)
- Перегляд результатів (відповіді з розбивкою)
- Видалення

---

### 3.12 Налаштування платформи (`/admin/settings`)

#### Загальні налаштування
- Назва застосунку, URL, email підтримки
- Часовий пояс
- Favicon (upload)

#### Налаштування платіжного шлюзу
- Вибір активного шлюзу: LiqPay / UaPay
- LiqPay: public key, private key
- UaPay: api key, merchant ID, URL
- Тестовий запит для перевірки підключення

#### Google OAuth
- Client ID та Client Secret (для Google Calendar OAuth)

#### reCAPTCHA
- Site key та Secret key

## 4. КАБІНЕТ СПЕЦІАЛІСТА

Доступний за `/admin/specialist/...` після входу користувача з роллю "specialist".

### 4.1 Дашборд спеціаліста (`/admin/specialist/dashboard`)

- Статистика: кількість записів сьогодні, цього тижня, завершених
- Список найближчих записів

### 4.2 Мої записи (`/admin/specialist/appointments`)

### 4.3 Мій розклад (`/admin/specialist/slots`)

- Сітка днів тижня + часові слоти
- Редагування власного розкладу (доступні дні та слоти)
- Блокування конкретних дат (відпустка)

## 5. ІНТЕГРАЦІЇ

### 5.1 Google Calendar

#### Підключення
- OAuth 2.0 flow: `GET /v1/google/auth-url` → redirect → `GET /v1/google/callback`
- Scopes: `calendar`, `calendar.events`, `userinfo.email`, `openid`
- Зберігання access_token, refresh_token, expires_at, google_email у таблиці `user_google_token`
- Автоматичне оновлення токену (refresh) при закінченні терміну

#### Відключення
- `POST /v1/google/disconnect`: revoke token + видалення з БД

#### Статус
- `GET /v1/google/status`: connected (bool) + google_email

#### Створення події
- Тригери: підтвердження запису (subscription/free) + після успішної оплати (card)
- Подія: summary, description з міткою "Wellbeing", start/end (Europe/Kiev), attendees, reminders
- Нагадування: email за 24 год + popup за 1 год
- Учасники: Google email користувача + wellbeing email + email спеціаліста
- Google Meet конференція (тільки для `communication_method = 'google_meet'`)
- Збереження `google_event_id` і `google_meet_link` у запису

#### Видалення події
- При скасуванні запису — видалення події з Google Calendar

#### Відображення подій (дашборд)
- `GET /v1/google/upcoming-events`: фільтрація по description (містить "Wellbeing")
- Повернення: id, title, start, end, meet_link, html_link
- Відображення у блоці дашборду з кнопкою "Meet"

### 5.2 Платіжна система

#### Підтримувані шлюзи
- **LiqPay** (Приватбанк)
- **UaPay**
- Активний шлюз конфігурується в адмін-налаштуваннях

#### Ініціювання оплати
- `POST /v1/payment/{id}/initiate`
- Створення order через API шлюзу
- Збереження `gateway_order_id`, `gateway`
- Повернення `checkout_url` → редирект на сторінку оплати

#### Синхронізація статусу
- `POST /v1/payment/{id}/sync` — polling за payment ID
- `POST /v1/payment/sync-by-order` — за `gateway_order_id` (при поверненні з checkout)
- Логіка: `success` → payment_status=completed, appointment=paid+confirmed, Google Calendar, email-сповіщення, Creatio sync
- Логіка: `failed` → payment_status=failed

#### Callback (webhook)
- `POST /v1/payment/callback/handle`
- Верифікація підпису шлюзу
- Обробка результату (аналогічно sync)

#### Повернення коштів (refund)
- При скасуванні оплаченого запису (card) — автоматичний refund через API шлюзу
- Статус: refunded / failed
- Email-сповіщення про повернення
- Термін: 3–5 робочих днів

#### Корпоративна підписка
- `payment_via = 'subscription'` → без транзакції, `payment_status = 'paid'`
- Рахується як безкоштовний сеанс з ліміту компанії (поточний місяць)

- Синхронізація при реєстрації

  - Назва, дата/час, учасники
- При оплаті: `markAppointmentPaid()` → оновлення Activity в Creatio

## 8. АРХІТЕКТУРА БЕКЕНДУ

### 8.1 API структура
- Базовий URL: `/api/v1/`
- Авторизація: `Authorization: Bearer {token}` для всіх захищених endpoints
- Формат: JSON (Content-Type: application/json)
- HTTP-методи: GET, POST, PUT, DELETE, OPTIONS (CORS preflight)

### 8.2 Модулі
- `api/modules/v1/controllers/` — основні контролери
- `common/models/` — ActiveRecord моделі
- `common/services/` — бізнес-логіка

### 8.3 Кешування
- Список спеціалістів: 60 секунд (ключ включає дату і годину)
- Портальні налаштування: 300 секунд

### 8.4 Безпека
- Bearer Token авторизація (JWT)
- CORS налаштований для frontend домену
- reCAPTCHA на публічних формах
- Валідація вхідних даних через Yii2 Model Rules
- Захист від повторного відгуку (409 Conflict)
- Перевірка доступу (user_id match) на кожен ресурс

## 9. ДЕПЛОЙ ТА ІНФРАСТРУКТУРА

### 9.1 CI/CD Pipeline (GitHub Actions)
1. Білд Angular (`npm ci && npm run build --configuration=production`)
2. Rsync до сервера (frontend)
3. SSH: `git pull` → `composer install --no-dev` → `php yii migrate --interactive=0` → restart systemd

### 9.2 Production сервер
- PHP systemd service (`wellbeing-api.service`)
- Caddy як reverse proxy (конфігурація у `Caddyfile`)
- PostgreSQL: Supabase cloud
- PHP_CLI_SERVER_WORKERS=4 (multi-threaded PHP built-in server)

### 9.3 Захищені файли (не комітяться у git)
- `backend/wellbeing-api/common/config/main-local.php` (DB credentials)
- `backend/wellbeing-api/common/config/params-local.php` (secrets: reCAPTCHA, Google OAuth, payment keys)
- `Caddyfile`

### 9.4 Проксі (local development)
- Angular Dev Server: `http://localhost:4200`
- PHP Backend: `http://127.0.0.1:8000`
- `proxy.conf.json` → target: `http://127.0.0.1:8000` (обов'язково IPv4, не localhost)
